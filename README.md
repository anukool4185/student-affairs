# ระบบจัดเก็บข้อมูลและรูปภาพนักเรียน — คู่มือฉบับง่าย (3 ไฟล์)

ชุดนี้มีแค่ **3 ไฟล์** เหมือนที่คุณคุ้นเคย ไม่มีไฟล์เสริมอะไรให้งงครับ:

| ไฟล์ | ไปวางที่ไหนใน Apps Script |
|---|---|
| `Code.gs` | ลบโค้ดเดิมในไฟล์ `Code.gs` ของโปรเจกต์ → วางทับ |
| `Index.html` | สร้างไฟล์ HTML ใหม่ชื่อ `Index` → วางเนื้อหา |
| `appsscript.json` | เปิดผ่าน Project Settings (ติ๊ก "Show appsscript.json") → วางทับ |

---

## ขั้นตอนติดตั้ง (ทำใน Browser ทั้งหมด ไม่ต้องลงโปรแกรม)

1. เปิด https://sheets.google.com → สร้าง Sheet ใหม่ ตั้งชื่อ เช่น "ระบบนักเรียน"
2. เมนู **ส่วนขยาย (Extensions) > Apps Script**
3. ในไฟล์ `Code.gs` ที่มีอยู่แล้ว → Ctrl+A ลบทั้งหมด → วางเนื้อหาจาก `Code.gs` ของชุดนี้
4. กดไอคอน **+** ข้าง "Files" → เลือก **HTML** → ตั้งชื่อ `Index` → วางเนื้อหาจาก `Index.html` ของชุดนี้
5. ไอคอนเฟือง **Project Settings** → ติ๊ก "Show appsscript.json manifest file in editor" → กลับไปที่ไฟล์นั้น → วางเนื้อหาจาก `appsscript.json` ของชุดนี้
6. Ctrl+S บันทึก
7. **Deploy > New deployment > Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
   - กด Deploy → Authorize → Advanced → Go to [ชื่อโปรเจกต์] (unsafe) → Allow
8. คัดลอก Web app URL ไปแจกผู้ใช้งาน

ทดสอบ login ด้วย `admin` / `admin123` ก่อน (แล้วอย่าลืมเปลี่ยนรหัสผ่านทันทีในแท็บ "บัญชีของฉัน")

---

## เก็บไว้บน GitHub แบบง่าย (ไม่ใช้ clasp)

1. เข้า https://github.com → กด **New** สร้าง repository ใหม่
2. ในหน้า repo กด **Add file > Upload files**
3. ลากไฟล์ `Code.gs`, `Index.html`, `appsscript.json` (และ `README.md` นี้) ขึ้นไป
4. กด Commit — เสร็จแล้ว เป็นแค่ที่เก็บโค้ดสำรอง ไม่เชื่อมต่อกับ Apps Script โดยตรง

เวลาจะอัปเดตโค้ดจริง: แก้ในไฟล์ที่ Apps Script editor ตามปกติ → ถ้าต้องการ ค่อยมา copy เนื้อหาที่แก้แล้วไปอัปเดตในไฟล์บน GitHub ด้วย (อัปโหลดทับไฟล์เดิมได้เลย)

## เวลาจะแก้ไขอะไร ต้องแก้ไฟล์ไหน

| ต้องการแก้... | แก้ไฟล์ |
|---|---|
| ข้อความ/สี/ปุ่ม/หน้าตา/ลำดับฟอร์ม | **Index.html** เท่านั้น |
| รายชื่อ "ระดับชั้น" และ "ห้อง" ที่ให้เลือก | **Index.html** เท่านั้น — หาคำว่า `gradeOptions` และ `roomOptions` ใกล้ต้นไฟล์ในส่วน `<script>` แก้รายการในนั้นได้เลย |
| เพิ่มฟิลด์ข้อมูลใหม่ (เช่น ชื่อผู้ปกครอง) ที่ไม่กระทบกฎสิทธิ์ | ส่วนใหญ่ Index.html, บางครั้งต้องแตะ Code.gs เพิ่ม 1 คอลัมน์ |
| เปลี่ยนกฎสิทธิ์/บทบาท/ขั้นตอนอนุมัติทั้งระบบ | **Code.gs** (ไม่ใช่เรื่องที่ต้องทำบ่อย) |

> **สำคัญ:** แก้โค้ดเสร็จแล้วต้อง **Deploy ใหม่** ทุกครั้ง ไม่งั้นลิงก์เดิมจะยังเป็นโค้ดเก่า — Deploy > Manage deployments > ไอคอนดินสอ > New version > Deploy

## บทบาทและสิทธิ์การใช้งาน (เหมือนเดิม)

| บทบาท | Username เริ่มต้น | Password เริ่มต้น | สิทธิ์ |
|---|---|---|---|
| แอดมิน | `admin` | `admin123` | ทำทุกอย่างได้ทันที + เผยแพร่รูป + อนุมัติคำขอจากครู + จัดการบัญชีครู/บริษัท |
| ครูที่ปรึกษา | `tadmin` | `tadmin1` | เพิ่ม/อัปโหลด/ยืนยันรหัส ทันที — ย้ายห้อง/ลบ ต้องส่งคำขอให้แอดมินอนุมัติ |
| นักเรียน | รหัสนักเรียนปัจจุบัน | เบอร์โทรนักเรียน | ดูข้อมูลตัวเอง แก้ได้แค่เบอร์โทร |
| บริษัทภายนอก | `company` | `company1` | ค้นหา + ติ๊กเลือก + ดาวน์โหลด (ZIP รูป+CSV) |

**เวิร์กโฟลว์รูป:** อัปโหลด (แอดมิน/ครู) → พักในโฟลเดอร์ชั่วคราว → แอดมินเข้าแท็บ "จัดการส่งข้อมูล" เลือกรายการ+วันที่(ถ้าต้องการ) → กด "เผยแพร่" → ไฟล์ไปอยู่โฟลเดอร์จริงตาม `[วันที่ถ้ามี]/ระดับชั้น/ห้อง`

มีจุดไหนติดขัดหรืออยากปรับเพิ่ม บอกได้เลยครับ
